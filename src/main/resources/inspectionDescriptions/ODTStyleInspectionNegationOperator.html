Inspects usage of collection operators EMPTY and EXISTS in combination with the NOT operator

Operators EMPTY and EXISTS are often used in combination with the NOT operator which is not necessary.
In general, it is good practice to use positive assertions over negative assertions because it makes the code more readable.
For example:
IF $value / NOT EMPTY { } ==> IF $value / EXISTS {}

Three scenarios for unnecessary negation are detected:
Leading negation: $value / NOT EMPTY
Inside negation: NOT($value / EMPTY)
Trailing negation: $value / EMPTY / NOT

For all three scenarios, a quickfix is available to replace the negative assertion with a positive assertion
